# 08. 원본 UI 정밀 조사 및 개편 설계

1차 프로토타입 UI가 원본과 차이가 크고 디테일이 부족하다는 피드백에 따라,
실제 Flow Production Tracking(구 ShotGrid/Shotgun) UI를 정밀 조사하고 개편 방향을 정한다.

> 조사 출처: Autodesk 공식 제품/기능 페이지, ShotGrid Knowledge Network,
> Flow Production Tracking 커뮤니티 포럼, python-api 문서 (하단 출처 목록).

---

## 1. 원본 UI의 구조적 특징 (정밀)

### 1.1 전역 크롬(Global Chrome)
- **상단 글로벌 바**: 좌측 제품 로고 → 프로젝트 스위처(드롭다운) → 글로벌 메뉴
  (Inbox/Activity, Media, My Tasks, People, Apps) → 우측 전역 검색 + 사용자 아바타 메뉴.
- **프로젝트 네비게이션**: 프로젝트 진입 시 그 프로젝트의 **페이지 탭/목록**
  (Overview, Shots, Assets, Sequences, Tasks, Versions, Notes, Playlists ...).
  사용자가 페이지를 자유롭게 만들고 핀 고정. → 본질적으로 **"엔티티 쿼리 = 페이지"**.

### 1.2 엔티티 페이지 = 쿼리된 그리드
원본 UI의 90%는 "필터된 엔티티 그리드"다. 한 페이지 = 한 엔티티 타입의 쿼리 결과.
- **상단 툴바**: 페이지명 + **결과 카운트**, `+ <Entity>` 새로 만들기 버튼,
  **Fields**(컬럼 표시/구성), **Filter**, **Group by**(그룹핑 필드),
  **Sort**, **display mode 토글**(List / Thumbnail / Detail / Card 등), 페이지 내 검색.
- **그룹핑(Group by)**: 행을 특정 필드로 묶고, **접을 수 있는 그룹 헤더 행**에
  그룹값 + **항목 수** + (옵션)컬럼 요약을 표시. 예: `SEQ010 (4)`, `In Progress (12)`.
  VFX 현업에서 시퀀스별/스텝별/상태별 그룹핑이 일상.
- **인라인 편집**: 셀을 더블클릭/클릭하여 그 자리에서 상태·담당자·값 변경.
- **표시 모드**:
  - *List View* — 데이터 집약형 스프레드시트(기본)
  - *Thumbnail View* — 썸네일 카드 격자(시각 집약형). 토글로 즉시 전환.

### 1.3 ★ 파이프라인 스텝 상태 요약 (Pipeline Step summary) ★
원본의 **가장 상징적인 UI 요소**. Shot/Asset 행에, 그 엔티티에 속한 **Task들을
공정(Step)별로 한 칸씩 색칠한 작은 셀 줄**로 표시한다.
- 각 셀 = 한 스텝(LAY/ANM/FX/LGT/CMP ...)의 Task 상태 색.
- 한눈에 "이 샷이 어느 공정까지 진행됐는지"를 본다.
- 공식 문서상 이 summary 필드(`step_0`, `step_13` 등)는 **UI 클라이언트에서 계산**됨.
→ 본 프로토타입은 백엔드 `pipeline` 필드(스텝별 status/color 배열)로 동일 재현.

### 1.4 상태(Status) 표현
- 상태는 **색상**이 핵심. 셀에 색을 입히거나 색 점+약어/이름으로 표시.
- 기본 상태 색(시드 반영): wtg 회색 / rdy 하늘 / ip 주황 / rev 라벤더 /
  apr 연두 / fin 진녹 / hld 빨강 / omt 진회.

### 1.5 미디어 리뷰 (Review / Screening Room)
- **Media 페이지**: Version 썸네일 격자. 상태/노트수/제출자 표시.
- 클릭 시 **플레이어 + 우측 노트/Activity 패널**. 프레임 주석(annotation),
  버전 비교, Playlist 재생(데일리). 노트는 답글 스레드 + 아바타.

### 1.6 디테일 페이지
- 상단 **엔티티 헤더**(큰 썸네일 + 핵심 필드). 하단/우측 **연관 탭**
  (Tasks / Versions / Notes) + **Activity 스트림**(시간순 변경/코멘트).

### 1.7 스케줄
- **간트 차트**: Task를 막대로, 드래그로 일정 조정, 스텝/우선순위 색.

## 2. 1차 프로토타입의 격차 진단

| 원본 요소 | 1차 구현 | 격차 |
|-----------|:--------:|------|
| 전역 글로벌 바 + 프로젝트 페이지 탭 | 단순 사이드바만 | 🔴 큼 |
| 그리드 툴바(카운트/+New/Group/Mode) | 없음 | 🔴 큼 |
| 행 그룹핑(접이식 헤더+카운트) | 없음(평면 테이블) | 🔴 큼 |
| **파이프라인 스텝 상태 스트립** | 없음(task_count 숫자만) | 🔴 큼(시그니처) |
| 표시 모드 토글(List/Thumbnail) | List만 | 🟡 |
| 상태 색 셀 | pill만(작음) | 🟡 |
| 리뷰 Screening Room | 노트만 | 🟡 |

## 3. 개편 설계 (2차)

### 3.1 전역 크롬
- 상단 **글로벌 바**: 로고 / 프로젝트 스위처 / 전역 페이지(Media·My Tasks) / 검색 / 아바타.
- 좌측 **프로젝트 페이지 네비**: Overview·Shots·Assets·Sequences·Review·Playlists,
  각 항목에 결과 카운트 뱃지.

### 3.2 EntityGrid (Shots/Assets 공용 컴포넌트)
- **툴바**: `Shots ▾  9 results | + Shot | Group by: [Sequence ▾] | View: [≣ List][▦ Thumb] | 🔍`
- **List 모드**: 컬럼 = 썸네일 / 코드 / 상태(색셀+인라인편집) / **Pipeline 스트립** /
  Cut In·Out·Dur / 정보. **Group by**로 시퀀스·상태별 **접이식 그룹 헤더 + 카운트**.
- **Thumbnail 모드**: 썸네일 카드(상태 테두리 색, 코드, 파이프라인 스트립 미니).
- **인라인 상태 편집** 유지(시그니처).

### 3.3 PipelineStrip 컴포넌트
- `pipeline[]`을 받아 스텝별 색 셀을 가로로 렌더. 각 셀에 step_code 약어 + 상태색.
- 툴팁: `ANM · In Progress`.

### 3.4 Screening Room (Review 개편)
- 좌: Version 썸네일 격자(상태 테두리). 우: 선택 버전 **플레이어 영역(썸네일 확대 +
  프레임 슬라이더 모형) + Activity 노트 스레드 + 작성**.

### 3.5 색/타이포
- 배경 #25282c 계열, 라인 조밀(행 28px), 코드 등폭, 상태색 셀, 액센트 #00A0E4(샷건 블루).

---

## 4. 백엔드 변경 (개편 지원)
- `shot_dict` / `asset_dict` 에 **`pipeline`** 필드 추가:
  `[{step_code, step_name, step_color, status, status_name, status_color, task_id}]`
  (스텝 순 정렬). → 프론트 PipelineStrip 의 데이터 소스.

## 5. 2차 개편 구현 결과 (완료)

설계대로 프론트엔드를 전면 개편하여 다음을 구현·검증(빌드 통과 + 헤드리스 렌더 확인)했다.

| 개편 항목 | 결과 |
|-----------|------|
| 전역 글로벌 바(로고/프로젝트 스위처+ACTIVE/Media·My Tasks/검색/아바타) | ✅ |
| 좌측 프로젝트 페이지 네비 + **결과 카운트 뱃지** | ✅ (Shots 9, Assets 6, Tasks 63, Versions 12) |
| 그리드 툴바(엔티티/카운트/+New/Group by/표시모드 토글) | ✅ |
| **행 그룹핑 + 접이식 그룹 헤더 + 카운트** | ✅ (Shots=Sequence별, Assets=Type별) |
| **파이프라인 스텝 상태 스트립** (LAY·ANM·FX·LGT·CMP / MOD·RIG·SURF) | ✅ 시그니처 재현 |
| 표시 모드 List ⇄ Thumbnail 토글 + 카드 격자 | ✅ |
| Screening Room(Media 카드 격자 + 플레이어 패널 + Activity 노트) | ✅ |
| 프로젝트 자동 선택(원본의 "마지막 프로젝트 기억" 모사) | ✅ |
| 상태 색/조밀 행/샷건 블루 액센트(#00a0e4) | ✅ |

검증: `npm run build`(tsc+vite) 0 에러. 헤드리스 Chrome 렌더 캡처로
Overview/Shots/Assets/Review 화면 모두 정상 표시 확인.

### 백엔드 변경
- `serializers.pipeline_summary()` 신설 → `shot_dict`/`asset_dict` 에 `pipeline[]` 추가.
- 라우터 list 엔드포인트에서 `stepmap` 1회 생성 후 전달(N+1 완화).
