## A. 체크박스 일괄선택 — 두 모델 합의(= 내 구현과 일치)
- 내부 선택 상태 `Set<number>`, **고정 체크박스 컬럼**(columns/cfg.order/Fields 에서 제외 → 재배치·숨김·리사이즈 불가, 폭 ~34px).
- **select-all = filtered(보이는) 행** 기준, header 체크박스 indeterminate(ref).
- 체크박스 클릭 `stopPropagation`(행 클릭/편집과 분리), 별도 하이라이트 클래스(`row-checked`).
- 일괄 액션 바: "N selected" + Status 일괄 변경(행별 PATCH) + Clear.
- (GPT-5.5 제안) 추후 백엔드 `PATCH /shots/bulk { ids, data }` 로 최적화 가능 → 현재는 행별 PATCH.

## B. Schedule/Gantt — 두 모델 합의
- **EntityGrid 와 분리된 별도 페이지**.
- 날짜는 `YYYY-MM-DD` → **UTC day-index**(`Date.UTC`)로 파싱(타임존 1일 밀림 방지) ← 둘 다 강조.
- **pixels-per-day** 스케일 + zoom(Day26/Week11/Month4), bar = 절대위치 `left=x(start), width=(due-start+1)*ppd`.
- 좌측 라벨 sticky + 우측 타임라인 가로 스크롤, 월/주 눈금, **today 라인**, 엔티티별 그룹.
- 백엔드는 denormalized task(step/status 색 포함) 반환 → 기존 `/api/tasks` 가 이미 그 형태라 재사용.
- 미스케줄(null 날짜) 분리 표기.

## 메인 에이전트 처리
- 위 합의안대로 구현 완료(체크박스 컬럼·일괄 Status 바 / Schedule 페이지·간트 바·줌·today).
- seed 의 task 날짜를 **공정별 staggered** 로 변경 → 계단식 간트가 실제로 보이게.
- 두 모델이 체크박스(거의 내 안과 동일)·간트(UTC 파싱·ppd·분리 페이지) 를 독립적으로 합의 → 신뢰 높음.
