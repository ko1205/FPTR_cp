## 질문 요지
상단 네비는 반응형 유지하되 **그리드는 고정폭(창 작아지면 가로 스크롤)**, **컬럼은 창에
반응하지 말고 마우스 드래그로 리사이즈**, **Description 등 텍스트 컬럼은 더블클릭 편집**,
**샷마다 task 수가 다른 파이프라인**을 어떻게 표현할지.

## 두 모델의 답변 — 거의 완전 합의

| 항목 | 합의 내용 |
|------|-----------|
| 1. 레이아웃/스크롤 | scroll wrapper `overflow:auto` + `<table table-layout:fixed; width:max-content; min-width:100%>` + `<colgroup><col width>` 픽셀폭. 셀 `overflow:hidden;text-overflow:ellipsis;white-space:nowrap`. sticky header(스크롤 컨테이너가 overflow 소유). **`width:100%` 금지**(컬럼 압축됨). |
| 2. 컬럼 리사이즈 | 직접 구현 권장(TanStack은 과함). `<th>` 우측에 그립 → `onPointerDown` + pointer capture/document 리스너로 `width = startWidth + dx`. 너비맵 `Record<key,number>` → `<colgroup>`에 적용(셀마다 적용 금지). localStorage 저장, min-width. **재배치 드래그와 충돌 → 그립에서 stopPropagation**. |
| 3. 더블클릭 편집 | `editingCell {rowId, colKey}` 상태. dblclick(stopPropagation) → input(autofocus). Enter/blur 저장, Escape 취소. **중복 저장 가드**(isSaving), 행선택과 분리. |
| 4. 파이프라인 | **둘 다 "step 정렬" 강력 권장.** 가변 strip은 "3번째 초록"이 행마다 다른 공정 → 스캔 불가. 표준 step 순서 정의 → `taskByStep` 맵 → 고정 서브컬럼, **없는 task는 빈 슬롯**. **step code(라벨 아님)로 정렬.** |

> 두 모델이 독립적으로 **거의 동일한 CSS/구현**을 제시 → 신뢰도 매우 높음.
> 특히 파이프라인은 내 기존(가변 strip)을 **둘 다 반대**, step 정렬로 전환 권고.

## 메인 에이전트(Claude)의 판단과 처리

채택 — **네 항목 모두 합의안대로 구현**:
1. `.grid-body` 스크롤 + `table-layout:fixed` + `<colgroup>` 픽셀폭 + `width:max-content`.
2. 컬럼 리사이즈: gridView 에 `widths` 추가(localStorage), `<th>` 그립 pointer 이벤트,
   재배치 드래그와 분리(grip stopPropagation), min 80px.
3. GridColumn 에 `edit` 옵션 추가 → Description 더블클릭 편집 → PATCH. Enter/Esc/blur, 가드.
4. PipelineStrip → **step 정렬**: 표준 shot step 순서(MM/LAY/ANM/FX/LGT/CMP/ROTO)로
   고정 슬롯, 없는 step 은 빈칸. step code 로 매칭.

추가: seed 를 **가변 파이프라인**(샷별 2~6 tasks)으로 변경 → 변화가 실제로 보이게.
