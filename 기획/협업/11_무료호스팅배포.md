## 핵심 트레이드오프 (Render vs HF Spaces)
- **Render**: 표준 git push→배포, **무카드**, 512MB, 15분 유휴→**30–60s 콜드스타트**, 750시간/월(1앱 적합).
- **HF Spaces**: **16GB RAM + 48h 슬립**(가끔 보는 데모에 훨씬 친화적), 설정 약간 더(Dockerfile + 포트 **7860**).
- Koyeb: 가능하나 카드 가홀드($29 취소) 가능성 + 리전 제한.

## 채택 결정
**Render 무료 Web Service + (A)단일서비스 + SQLite 부팅재시드 + 상대경로 API.**
- 이유: 무카드·표준 흐름·동일오리진(CORS 0)·우리 dev 프록시 구조와 1:1 매핑. 콜드스타트는 데모 허용범위.
- 대안: 가끔 접속·여유 RAM 원하면 **HF Docker Space**(48h 슬립). 영속 데이터 필요해지면 **Neon Postgres**.

## 구현 시 필요한 변경 (배포용, 미실행 — 사용자 승인 후)
1. **FastAPI가 빌드된 SPA 서빙**: `app.mount` 로 `dist` 정적 + **catch-all → index.html**(SPA 라우팅), 기존 `/api`·`/static` 유지.
   주의: catch-all 이 `/api`·`/static` 보다 **나중에** 등록되어야 함.
2. **프론트 상대경로화**: `client.ts` 의 API base 가 prod 에서 `/api` (현재 dev 프록시라 이미 상대일 가능성 → 확인 필요).
3. **부팅 재시드**: `main.py` lifespan/startup 에서 projects 0건이면 `seed.py` 로직 실행(이중시드 가드).
4. **컨테이너화**: 멀티스테이지 `Dockerfile`(node 스테이지 `npm ci && npm run build` → python 스테이지 `pip install` + `dist`/`static` 복사), `uvicorn app.main:app --host 0.0.0.0 --port $PORT`. (또는 Render Build/Start Command 2줄.) **$PORT 환경변수 바인딩 필수.**
5. mp4 5개(~5MB)·썸네일 **git 커밋 유지**(이미 커밋됨). LFS/외부스토리지 불필요.
6. **GitHub 원격 추가 후 push** → Render 연결.
   - 폴더 구조 주의: 코드가 `개발/backend`, `개발/frontend` 에 있고 requirements.txt 위치/한글 경로를 Docker/Render 설정에서 정확히 지정.
