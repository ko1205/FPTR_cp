# Free Hosting Research — FastAPI + SQLite + Vite SPA Demo (2026-06-27, Claude general-purpose 서브에이전트, 웹 검증)

> 단일 개발자 데모, 데이터 휘발 OK(부팅 재시드), ~5MB 정적자산(mp4 5 + 썸네일), 160KB SQLite.

## 2. 아키텍처 옵션
- **(A) 단일 서비스** — FastAPI가 `/`(빌드된 SPA, `StaticFiles(dist, html=True)`+catch-all→index.html) + `/api` + `/static`. **장점**: 1배포, **동일 오리진→CORS 없음**, API base URL 고민 제거, 무료 1앱 한도에 적합. **단점**: 프론트 변경 시 백엔드 재배포, 함께 슬립. → **Render/HF/Koyeb에 권장 형태.**
- **(B) 분리** — SPA(Cloudflare/Netlify/Vercel) + API(Render/HF/Koyeb). **장점**: SPA는 상시 CDN. **단점**: **2서비스+CORS+빌드시 API URL 주입**, 데모엔 과함.

## 3. 휘발 FS의 SQLite
- Render/HF/Koyeb 무료 = **컨테이너 FS 휘발**: `ftpr.db` 런타임 쓰기는 재배포/재시작/슬립복귀 시 소실. (HF 무료는 /data 유료만 영속, Render 무료 영속디스크 없음.)
- **이 앱엔 OK** — 데이터 폐기 전제 + 시드 존재. **부팅 재시드**(lifespan/startup에서 비어있으면 seed) 가 가장 단순·정확.
- 영속 필요 시: **Neon free**(0 스케일, $5 캡, 재시작 생존) 가 최선. Supabase=7일 무활동 일시정지, Render Postgres=30일 만료(데모 부적합).
- **결론: SQLite + 부팅재시드 유지.** 영속 원하면 Neon.

## 4. 함정
- 콜드스타트: Render 15분→30–60s, HF 48h(데모 친화적), Koyeb 1h. keep-alive 핑은 Render 750시간 소진/취지 위반 → 데모는 그냥 수용.
- 빌드: 프론트 `npm ci && npm run build`→dist; 백엔드 `pip install -r requirements.txt`; start `uvicorn app.main:app --host 0.0.0.0 --port $PORT`. **$PORT 바인딩**(HF Docker는 기본 **7860**/`app_port`).
- CORS: 분리 시에만 필요 → 단일서비스가 회피.
- mp4 ~5MB: **git 커밋 OK**(LFS/외부스토리지 불필요, HF 25MB/파일 제한 미만).
- 프론트 prod API URL: 단일서비스면 **상대경로**(`/api`,`/static`)로 끝. 분리면 `VITE_API_BASE` 빌드시 주입.

## 5. 추천
**최선 = Render 무료 Web Service(또는 HF Docker Space), 아키텍처 (A) 단일서비스 + SQLite 부팅재시드 + 상대경로.**
- **Render**: 표준 git push→web service, 무카드, 15분 후 30–60s 콜드스타트 감수. 1앱이 750시간에 적합.
- **HF Spaces**: **16GB RAM + 48h 슬립**(가끔 보는 데모에 이상적), 설정 약간 더(Dockerfile+7860).
- 회피: Fly(무료X)/Railway(트라이얼)/PythonAnywhere(ASGI 부적합).

### Render 단계 개요
1. FastAPI가 SPA 서빙(StaticFiles dist html=True + catch-all→index.html), `/api`·`/static` 유지, 프론트 **상대경로**화.
2. startup 재시드: projects 없으면 seed 실행.
3. 멀티스테이지 Dockerfile(node로 dist 빌드 → python로 reqs+dist+static 복사), `uvicorn ... --port $PORT`. mp4 커밋.
4. GitHub push.
5. Render: New→Web Service→repo 연결→Free→deploy(무카드).
6. `.onrender.com` 접속, 최초 30–60s 콜드스타트 후 시드된 데모 로드.

### HF Spaces 변형
동일 앱변경 + Docker Space, **7860** 리슨(또는 app_port), mp4 커밋 → 16GB/48h.
