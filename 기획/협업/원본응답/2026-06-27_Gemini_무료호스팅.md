1. **Hosts (Single-Service is best):**
   * **Render:** Free Web Service (512MB RAM, spins down after 15m inactivity).
   * **Koyeb:** Free Web Service (512MB RAM, low cold-start latency).
   * **Railway:** Free trial tier (ephemeral, easy setup).
2. **Database:** **SQLite with re-seed-on-boot** is simplest. Run your database creation and seeding scripts dynamically inside FastAPI's `@app.on_event("startup")` handler to ensure the demo has data after every container restart.
3. **Gotchas:** 
   * **Cold Starts:** Render takes ~50s to boot on first request after idling.
   * **Ephemeral File System:** SQLite changes will wipe on container restart/sleep.
   * **Base URL:** Ensure frontend API requests use relative paths (`/api/...`) since the backend serves the frontend.
4. **Recommended Path (Render Single-Service):**
   * **Step 1:** Add a build script that runs `npm run build` in the frontend and copies the `/dist` output folder into the backend static folder.
   * **Step 2:** In FastAPI, mount the static folder: `app.mount("/", StaticFiles(directory="dist", html=True), name="static")`.
   * **Step 3:** Add an on-startup db seed function in FastAPI.
   * **Step 4:** Push to GitHub and deploy to Render using Python runtime with start command `uvicorn app.main:app --host 0.0.0.0 --port $PORT`.
