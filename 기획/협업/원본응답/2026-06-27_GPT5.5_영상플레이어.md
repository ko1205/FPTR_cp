- **1. Frame-Accurate Stepping**
  - Clean approach: pause the video, then seek by frame duration: `video.currentTime = clamp(video.currentTime + direction * (1 / fps), 0, video.duration)`.
  - FPS: prefer `fps = version.frame_count / video.duration` after `loadedmetadata`; fall back to `24` if metadata/duration is unavailable or suspicious. If your source data later has explicit FPS, use that instead.
  - `requestVideoFrameCallback`: useful for reading the actual presented frame after decode or updating overlays, but it does not magically make seeking frame-accurate. Use it to confirm/display after a seek, not as the primary stepping mechanism.
  - Pitfalls: browser seeking is time-based, not true frame-index based; MP4 keyframes affect seek latency; `currentTime` may land near a frame, not exactly on it; floating-point drift accumulates, so compute target from rounded frame index when possible: `targetFrame = round(currentTime * fps) + dir`.

- **2. React Player Structure**
  - Use refs for the `<video>` element and imperative controls: play, pause, stop, seek, frame-step.
  - Use state only for UI display: `isPlaying`, `duration`, `fps`, `currentFrame`, `frameCount`.
  - Listen to `loadedmetadata`, `play`, `pause`, `ended`, and a throttled `timeupdate`.
  - For live frame counter: `Math.round(video.currentTime * fps)`, clamped to `0..frame_count - 1`.
  - Avoid setting React state on every raw `timeupdate` if the displayed frame did not change. Compare previous frame first, or throttle with `requestAnimationFrame`.

- **3. Keyboard Scope**
  - Use a `useEffect` keydown listener only while the review panel is open / active.
  - Document-level listener is fine if guarded carefully; requiring focus on the player is stricter but less convenient.
  - Guard typing targets:
    ```ts
    const el = document.activeElement;
    const typing =
      el instanceof HTMLInputElement ||
      el instanceof HTMLTextAreaElement ||
      (el as HTMLElement | null)?.isContentEditable;
    ```
  - On `ArrowLeft` / `ArrowRight`, `preventDefault()`, pause, then step one frame. Also ignore with `event.metaKey`, `ctrlKey`, or `altKey`.

- **4. FastAPI / StaticFiles Range Streaming**
  - Verify browser requests get:
    - `Accept-Ranges: bytes`
    - `Content-Type: video/mp4`
    - `206 Partial Content` for range requests
    - valid `Content-Range`
  - Test seeking in DevTools Network, not just initial playback.
  - Pitfalls: wrong MIME type, missing range support behind a proxy, MP4 not optimized for streaming. Ideally encode with metadata/moov atom at front, e.g. `ffmpeg -movflags +faststart`.
  - Starlette `StaticFiles` should support range responses in current versions, but verify with:
    ```bash
    curl -I -H "Range: bytes=0-1023" http://127.0.0.1:8000/static/media/sample.mp4
    ```
