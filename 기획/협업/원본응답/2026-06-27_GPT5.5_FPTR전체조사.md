## Prioritized Recommendations

1. Add real detail routes and full-page detail layout.
   - Files: `App.tsx`, new `pages/EntityDetail.tsx`, replace or demote `EntityDetailPanel.tsx`.
   - Implement routes like `/detail/:entityType/:id`; build breadcrumb, prev/next, large thumbnail, field table, and tabs for `Activity`, `Shot Info`, `Tasks`, `Notes`, `Versions`, `Publishes`, `Assets`, `History`.

2. Complete entity grid chrome and selection model.
   - Files: `EntityGrid.tsx`, `styles.css`.
   - Add leading checkbox column, row selection state, refresh button, layout button, active Pipeline dropdown, and make Add/Sort/Filter real instead of disabled.

3. Expand default Shot/Asset/Version columns.
   - Files: `Shots.tsx`, `Assets.tsx`, backend `models.py`/`schemas.py`/`serializers.py`.
   - Add modeled fields for Reel, Work Guide, Priority, Camera, Lens, Type, Tags, Note summary; show Assets as linked teal entity chips. Keep blanks visible instead of substituting too many placeholders.

4. Treat pipeline summary as optional field/view, not hardcoded default.
   - Files: `Shots.tsx`, `Assets.tsx`, `EntityGrid.tsx`, `gridView.ts`.
   - Keep `PipelineStrip`, but hide/show through Fields/Pipeline menu and persist per view.

5. Build a real Versions entity grid, then layer Review on top.
   - Files: new/reworked `pages/Versions.tsx`, `Review.tsx`, `ProjectTabs.tsx`.
   - Use `EntityGrid` for Versions with Status, Thumbnail, Version Name, Link, Artist, Task, Notes, Date; keep player/review panel as selected media overlay/panel.

6. Add scheduling/gantt/calendar view.
   - Files: new `pages/Schedule.tsx`, `ProjectTabs.tsx`, backend `routers/tasks.py`.
   - Use existing task `start_date`/`due_date`; add drag/update endpoints later. Real CSS evidence points to calendar/gantt components, so this is a major missing product surface.

7. Upgrade custom fields from demo fields to FPTR-like field definitions.
   - Files: `models.py`, `schemas.py`, `routers/custom_fields.py`, `EntityGrid.tsx`.
   - Add field data type variants: text, number, checkbox, date, list, entity link, multi-entity, status/list, URL. Add display order, hidden/default visibility, required/read-only, and per-project/entity scope.

8. Add activity/history model.
   - Files: backend new `EventLog`/`History` model, serializers, detail page.
   - Real detail pages include Activity and History; current notes are not enough to represent field changes, status changes, publishes, and version submissions.
