## 두 모델 합의 (거의 동일)
1. **프레임 이동**: pause 후 **정수 프레임 인덱스에 앵커링**해 seek —
   `target=clamp(round(currentTime*fps)+dir,0,total); video.currentTime=target/fps`.
   (`+=1/fps` 반복은 부동소수 드리프트로 프레임 경계를 벗어남.) `requestVideoFrameCallback`은
   stepping 정확도를 보장하지 않으니 표시 동기화용으로만.
2. **React 구조**: video는 **ref로 명령제어**, state는 **isPlaying만**. 프레임 카운터는
   `timeupdate/seeked`에서 **DOM 직접 갱신**(ref.textContent) — timeupdate→state는 re-render 폭주.
   이벤트: loadedmetadata/play/pause/ended/timeupdate/seeked.
3. **키보드 ←/→**: `useEffect` window keydown, **입력(input/textarea/contenteditable) 중엔 무시**,
   meta/ctrl/alt 무시, preventDefault. (focus 의존보다 scoped global + 입력 가드가 견고.)
4. **Range 스트리밍**: Accept-Ranges/206/Content-Type 확인, `ffmpeg -movflags +faststart` 권장.
   - 함정(둘 다): **Long-GOP 키프레임** 때문에 P/B 프레임 step 시 디코드 지연/스터터 →
     이상적으론 intra-only(g=1) 인코딩.

## 메인 에이전트 처리(완료)
- 백엔드: 실제 샘플 mp4 5개 다운로드(`app/static/media/v1~5.mp4`, test-videos.co.uk/MDN/w3schools),
  StaticFiles 로 서빙(content-type video/mp4). seed 의 version `media_url`을 실제 경로로.
- 프론트: `VideoPlayer`(Review.tsx) — `<video>` + 컨트롤(⏮ 이전프레임 / ▶⏸ 재생·일시정지 /
  ⏹ 정지 / ⏭ 다음프레임) + 프레임 카운터(DOM 갱신) + **키보드 ←→ 프레임이동, Space 재생**.
  프레임 이동은 정수 인덱스 앵커링(fps=24). poster=썸네일.
- 검증: 빌드 0 에러, video/mp4 200 서빙, 리뷰 패널에 실제 영상 프레임 렌더 캡처.
