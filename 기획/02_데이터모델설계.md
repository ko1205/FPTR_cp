# 02. 데이터 모델 설계

원본 FPTR의 엔티티 구조(`01` 문서)를 SQLite + SQLAlchemy ORM으로 옮긴 설계.
이 문서가 **백엔드·프론트엔드 공통 스펙**이다.

---

## 1. 설계 원칙

1. **원본 엔티티명/필드명 존중** — 가능한 한 FPTR 표준 명칭을 따른다
   (예: `cut_in`, `cut_out`, `sg_status_list` 개념 → 본 구현은 `status` 코드).
2. **상태(Status)는 코드 + 색상** — FPTR의 `sg_status_list`를 별도 테이블로.
3. **다대다는 연결 테이블** — Shot↔Asset, Task↔User, Playlist↔Version.
4. **Note는 다형 연결** — `link_entity_type` + `link_entity_id` 컬럼으로 단순화.
5. **소프트하게, 그러나 과하지 않게** — 프로토타입이므로 Custom Field 동적 스키마는
   제외하고 고정 컬럼으로 핵심만 표현.

## 2. 엔티티 정의

### 2.1 Status (상태 마스터)
| 컬럼 | 타입 | 설명 |
|------|------|------|
| code | TEXT PK | `wtg`, `rdy`, `ip`, `rev`, `fin`, `hld`, `omt`, `apr` |
| name | TEXT | 표시명 (Waiting to Start 등) |
| color | TEXT | HEX 색상 (#rrggbb) |
| sort_order | INT | 정렬 |

### 2.2 Step (파이프라인 공정)
| 컬럼 | 타입 | 설명 |

| id | INT PK | |

| color | TEXT | HEX |
| entity_type | TEXT | 적용 대상 'Asset'/'Shot' (참고용) |

### 2.12 연결(다대다) 테이블
- `shot_asset_link (shot_id, asset_id)` — 샷에 등장하는 에셋
- `task_assignee (task_id, user_id)` — 태스크 담당자
- `playlist_version (playlist_id, version_id, sort_order)`

## 3. ER 다이어그램 (텍스트)

```
                ┌─────────┐
                │ Project │
                └────┬────┘
       ┌────────────┼───────────────┬──────────────┐
       │            │               │              │
   ┌───▼────┐   ┌───▼───┐       ┌───▼───┐     ┌────▼────┐
   │Sequence│   │ Asset │       │ Shot  │     │Playlist │
   └───┬────┘   └───┬───┘       └───┬───┘     └────┬────┘
       │            │               │              │
       └──*Shot     │   shot_asset_link(다대다)     │playlist_version
                    │◄──────────────┤              │ (다대다)
                    │               │              ▼
                ┌───▼───────────────▼───┐      ┌────────┐
                │        Task           │      │Version │◄── user
                │ (entity_type/id 다형)  │──1:*─►│        │
                └───┬───────────────────┘      └───┬────┘
        task_assignee│(다대다)                      │
                ┌───▼─────┐                         │
                │HumanUser│                         │
                └─────────┘     Note(다형 link)─────┴──► (Version/Shot/Task/Asset)
```
