# 06. 화면 설계 (UI 구성)

원본 FPTR의 "사이드바 + 거대한 데이터 그리드 + 미디어 리뷰" 패턴을 재현한
프론트엔드 화면 설계.

---

## 1. 전체 레이아웃

```
┌───────────────────────────────────────────────────────────┐
│ Header:  [FPTR Copy]   프로젝트: [Project Nebula ▼]         │
├──────────┬────────────────────────────────────────────────┤
│ Sidebar  │  Main Content (라우트별 화면)                    │
│  ▸ Home  │                                                 │
│  ▸ Shots │   ┌── 엔티티 그리드 / 카드 그리드 / 리뷰 ──┐    │
│  ▸ Assets│   │                                        │    │
│  ▸ MyTask│   │                                        │    │
│  ▸ Review│   └────────────────────────────────────────┘    │
│  ▸ Plylst│                                                 │
└──────────┴────────────────────────────────────────────────┘
```

- **다크 테마**, 조밀한 행 높이, 코드값은 등폭 글꼴, 상태는 색상 pill.
- 선택된 프로젝트는 `localStorage` + React Context 로 전 화면 공유.

---

## 2. 라우트 / 화면 목록

| 경로 | 화면 | 핵심 컴포넌트 |
|------|------|--------------|
| `/` | **Dashboard** — 프로젝트 카드 + 통계(카운트, 상태분포 바) | Dashboard.tsx |
| `/shots` | **Shots 그리드** — 스프레드시트형, 인라인 상태편집 + 디테일 드로어 | Shots.tsx, EntityDetailPanel |
| `/assets` | **Assets 그리드** — 동일 패턴 | Assets.tsx |
| `/my-tasks` | **My Tasks** — 사용자 선택 → 할당 태스크, 인라인 상태편집 | MyTasks.tsx |
| `/review` | **Review/Media** — 버전 카드 그리드 → 리뷰 패널(노트 스레드) | Review.tsx |
| `/playlists` | **Playlists** — 플레이리스트 → 버전 필름스트립 | Playlists.tsx |

## 3. 공용 컴포넌트

| 컴포넌트 | 역할 |
|----------|------|
| `StatusBadge` | `status_color`/`status_name` 으로 색상 pill |
| `StatusSelect` | 클릭 시 상태 드롭다운 → PATCH 호출 (**시그니처 인터랙션**) |
| `StepBadge` | 파이프라인 스텝 색상 배지 |
| `Avatar` | 사용자 이니셜 + `color` 원형 아바타 |
| `Thumbnail` | `thumbnail` hex 색을 배경으로 한 썸네일 박스 |
| `Sidebar` / `Header` | 내비 + 프로젝트 스위처 |
| `EntityDetailPanel` | 우측 드로어, Tasks/Versions/Notes 탭 |

## 4. 시그니처 인터랙션 — 인라인 상태 편집

원본 FPTR에서 코디네이터/아티스트가 하루에 수십 번 하는 동작을 재현:

```
[그리드 셀]  In Progress(주황 pill)  ── 클릭 ──▶  상태 드롭다운 펼침
                                                  ├ Waiting to Start
                                                  ├ Ready
                                                  ├ ● In Progress
                                                  ├ Pending Review  ← 선택
                                                  └ ...
   선택 즉시  PATCH /api/{shots|assets|tasks}/{id} {status:"rev"}
            → TanStack Query 무효화 → 행 갱신
```

이 인터랙션을 Shots / Assets / My Tasks 세 화면에서 동일하게 제공한다.

---

## 5. 데이터 흐름

```
React 컴포넌트
   │  useQuery / useMutation (TanStack Query)
   ▼
src/api/client.ts  (fetch 래퍼)
   │  /api/...  (Vite proxy)
   ▼
FastAPI (127.0.0.1:8000)  ──  SQLAlchemy  ──  ftpr.db (SQLite)
```

- 모든 읽기는 `useQuery`, 변경은 `useMutation` 후 관련 쿼리 `invalidate`.
- 타입은 `src/api/types.ts` 에 인터페이스로 정의(엔티티 응답 필드 일치).
