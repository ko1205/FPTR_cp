# 07. 구현 결과 — 원본(FPTR) 대비 비교

프로토타입 완성 후, 원본 Flow Production Tracking과 **기능적 동등성**을 정량/정성
비교한다. (사용자가 "샘플과 문서를 비교"할 수 있도록 작성)

## 1. 데이터 모델 커버리지

| 원본 표준 엔티티 | 프로토타입 | 비고 |
|------------------|:---------:|------|
| Project | ✅ | name/code/status/기간 |
| Sequence | ✅ | |
| Shot | ✅ | cut_in/out/duration, 썸네일 |
| Asset | ✅ | asset_type 분류 |

| HumanUser | ✅ | role/department |

## 2. 워크플로우 재현도

| 흐름 | 원본 | 프로토타입 | 상태 |
|------|------|-----------|:----:|
| **Track** — Shot/Asset 상태 추적 | 그리드 인라인 편집 | 동일 패턴 구현 | ✅ |
| **Track** — Task 할당/상태 | 담당자 + 상태 워크플로우 | 구현(PATCH) | ✅ |
| **Review** — Version 검토 | 미디어 플레이어 + 노트 | 썸네일 + 노트 스레드 | 🟡 부분 |
| **Review** — Playlist(데일리) | 묶음 리뷰 | 필름스트립 | ✅ |
| **Manage** — 대시보드/진척 | 위젯/번다운 | 카운트 + 상태분포 바 | 🟡 부분 |
| **Manage** — My Tasks | 개인 할 일 | 구현 | ✅ |

🟡 = 개념·데이터는 동등하나, 실제 미디어 재생/고급 시각화는 placeholder.

## 3. API 동등성 (shotgun_api3 대비)

| 원본 패턴 | 프로토타입 | 동등성 |
|-----------|-----------|:------:|
| `find / find_one` | `GET /api/{e}` , `GET /api/{e}/{id}` | ✅ 의미 동일 |
| `create` | `POST /api/{e}` | ✅ |
| `update` | `PATCH /api/{e}/{id}` | ✅ |
| `delete` | `DELETE /api/{e}/{id}` | ✅ |
| 필터(filters) | 쿼리 파라미터(project_id/status/sort 등) | 🟡 단순화 |
| 필드 선택(fields) | 항상 enriched 전체 반환 | 🟡 미구현 |
| 썸네일 업로드 | hex 색상 placeholder | ⛔ |

→ **CRUD 의미 동등성은 100% 달성**. 임의 필터 표현식/필드 셀렉션은 단순화.

## 4. 명백한 갭 (의도적 미구현)과 이유

| 갭 | 이유 | 원본에서의 중요도 |
|----|------|------------------|
| DCC Toolkit(Maya/Nuke 통합) | 웹/도메인 검증이 목적, 범위 밖 | ★★★ (원본 핵심 강점) |
| 실제 미디어 트랜스코딩/재생 | 인프라 필요, placeholder로 충분 | ★★☆ |
| 권한관리(RBAC)/인증 | 프로토타입 단일 사용자 가정 | ★★☆ |

| 실시간 협업(WebSocket) | 단일 검증으로 충분 | ★☆☆ |
| Event Daemon / Webhooks | 자동화 범위 밖 | ★★☆ |
| 간트/스케줄 자동계산 | 날짜 필드만 보유 | ★☆☆ |

## 5. 결론 — "어느 정도까지 같게 만들 수 있는가"

1. **도메인 모델과 핵심 워크플로우(Track/Review/Manage)는 충실히 재현 가능**하다.
   엔티티 구조·상태 워크플로우·인라인 편집·리뷰 노트는 프로토타입 수준에서
   원본과 기능적으로 거의 동등하게 동작한다.

2. **원본의 진짜 해자(moat)는 도메인 모델이 아니라 "확장성"** 이다:
   - DCC 파이프라인 통합(Toolkit)
   - Custom Field 동적 스키마(스튜디오별 무한 커스터마이즈)
   - 이벤트 기반 자동화 / 검증된 전문 리뷰 도구(RV)
   이 부분이 프로토타입과 상용 제품의 핵심 격차다.

3. **검증 결과**: 핵심 추적/리뷰/관리 기능은 소규모 팀이 쓸 수 있는 수준까지
   재현 가능. 다만 "스튜디오 파이프라인 플랫폼"으로서의 가치는 통합·확장 레이어에
   있으며, 그 재현은 본 프로토타입 범위를 훨씬 넘어선다.
