# FTPR Copy — 프로젝트 개요

## 1. 목적

본 프로젝트는 **Autodesk Flow Production Tracking (FPTR)** 의 핵심 기능을 모사하는
**기능 검증용 프로토타입**이다.

- 정식 명칭(현재): **Flow Production Tracking** (Autodesk, 2024~)
- 이전 명칭: **ShotGrid** (2021~2023, Autodesk 인수 후)
- 최초 명칭: **Shotgun** (Shotgun Software, 2006~2021)

> 본 문서 전체에서 원본 제품을 통칭할 때 **FPTR** 또는 "원본"이라고 표기한다.

### 1.1 이 프로토타입으로 확인하려는 것

> "원래 제품과 기능적으로 어느 정도까지 같게 만들 수 있는가"

따라서 목표는 **상용 수준의 완성도**가 아니라, FPTR의 **핵심 도메인 모델과 워크플로우를
재현**하여 기능적 동등성의 한계와 가능성을 검증하는 것이다.

## 2. 핵심 제약 조건 (사용자 요구)

| 제약 | 적용 방식 |
|------|-----------|
| Python 패키지 global install 금지 | `개발/backend/venv` 로컬 가상환경만 사용 |
| TypeScript 패키지 폴더 내 격리 | `개발/frontend/node_modules` 로컬 설치만 사용 |
| 설치형 DB 사용 금지 | `개발/database/ftpr.db` — **SQLite 파일 DB** |
| 대분류 폴더 분리 | `기획/` (문서), `개발/` (코드) |
| 언어 혼용 시 명시적 분리 | `개발/backend`(Python), `개발/frontend`(TypeScript) 디렉토리 분리 |
| 기획 문서에 조사·방향·이유 상세 기록 | 본 `기획/` 폴더 전체 |

## 3. 폴더 대분류 구조

```
FTPR_copy/
├── 기획/                         # 기획·조사·설계 문서 (이 폴더)
│   ├── 00_프로젝트개요.md
│   ├── 01_연구조사_원본제품분석.md   # FPTR 도메인/기능 조사

│   ├── 03_기술스택선정이유.md        # 왜 이 스택인가
│   ├── 04_기능범위_로드맵.md         # 무엇을 구현/제외했나
│   ├── 05_API설계.md                # REST API 명세
│   ├── 06_화면설계.md               # UI/화면 구성
│   ├── 07_구현결과_원본비교.md       # 완성 후 원본과 비교
│   ├── 연구자료/                    # 1차 조사 메모

    ├── backend/                  # Python (FastAPI) — REST API 서버
    │   ├── venv/                 # 로컬 가상환경 (git 제외)
    │   ├── app/                  # 애플리케이션 코드

        └── ftpr.db
```

## 4. 언어 분리 원칙

서로 다른 언어는 `개발/` 하위에서 **물리적 디렉토리로 명시 분리**한다.

- **Python** 영역 → `개발/backend/` 안에서만 존재. venv도 이 안에 둔다.
- **TypeScript** 영역 → `개발/frontend/` 안에서만 존재. node_modules도 이 안에 둔다.
- 두 영역은 **HTTP REST API**로만 통신한다 (느슨한 결합).

## 5. 성공 기준 (Definition of Done)

1. `개발/backend`에서 venv 기반 FastAPI 서버가 기동된다.
2. `개발/frontend`에서 Vite 개발 서버가 기동되어 브라우저 UI가 뜬다.
3. SQLite 파일 DB에 샘플 프로젝트/샷/태스크/버전 데이터가 들어간다.
4. UI에서 다음 핵심 워크플로우가 동작한다:
   - 프로젝트 목록/생성
   - 샷·에셋 목록 (스프레드시트형 그리드)
   - 태스크 할당 및 상태 변경
   - 버전(미디어) 등록 및 리뷰 노트 작성
5. 한 번의 실행 스크립트로 전체 기동이 가능하다.

자세한 기능 범위는 `04_기능범위_로드맵.md` 참조.
