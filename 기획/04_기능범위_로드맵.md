## 2. 제외 범위 (Out of Scope) — 의도적 비구현

| 제외 기능 | 이유 |
|-----------|------|
| DCC 통합 (Maya/Nuke/Houdini Toolkit) | 프로토타입은 웹/도메인 검증이 목적 |
| 실시간 동시편집 (WebSocket) | 단일 사용자 검증으로 충분 |
| 권한관리(RBAC)/인증 | 프로토타입 — 모든 사용자 'admin' 가정 |
| Custom Field 동적 스키마 | 고정 컬럼으로 핵심 재현, 동적 스키마는 별도 과제 |
| 실제 미디어 트랜스코딩/스트리밍 | 썸네일·placeholder로 대체 |
| 간트/스케줄 자동 계산 | 날짜 필드만, 시각화는 선택 |
| 이메일/알림 | 범위 밖 |

→ 제외 항목은 **"원본 대비 미구현 갭"**으로 `07_구현결과_원본비교.md`에 정량 기록한다.

## 4. 기능적 동등성 평가 축

완성 후 아래 축으로 원본과 비교 (→ `07` 문서):
1. **데이터 모델 커버리지** — 원본 표준 엔티티 중 몇 %를 재현했나
2. **워크플로우 재현도** — Track/Review/Manage 각 흐름이 동작하나
3. **UI 패턴 재현도** — 그리드 인라인 편집, 상태 색상, 리뷰
4. **API 동등성** — 엔티티 CRUD 패턴이 shotgun_api3와 유사한가
5. **명백한 갭** — 무엇이 빠졌고 왜인가
