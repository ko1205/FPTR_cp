# 09. 원본 UI 실제 캡처 분석 (실측)

사용자의 **실제 Flow Production Tracking 인스턴스**(`4th-vfx-2023-1.shotgrid.autodesk.com`,
프로젝트 "CTR")를 헤드리스 브라우저로 **직접 캡처**하여 분석했다.
캡처 원본: [`이미지/원본참고/`](이미지/원본참고/)

| 파일 | 화면 |
|------|------|
| `00_홈_프로젝트목록.png` | 사이트 홈 = Projects 그리드 |
| `01_Projects_그리드.png` | Projects 엔티티 그리드(고해상도) |
| `02_Shots_그리드.png` | 프로젝트 내 Shots 그리드 |
| `03_Shot_상세페이지.png` | Shot 디테일 페이지 |
| `04_Versions_미디어그리드.png` | Versions(미디어) 그리드 |

> 캡처 방법: 사용자 Chrome 프로파일(인증 세션)로 `--headless --screenshot` 실행.
> 안정 URL 패턴 발견: `/page/project_default?entity_type=<Shot|Asset|Task|Version|...>&project_id=<id>`,
> 디테일은 `/detail/<EntityType>/<id>`.

## 1. 전역 구조 — **2단 상단 네비게이션** (중요)

원본은 좌측 사이드바가 아니라 **상단 가로 네비 2단**이다.

### 1단: 글로벌 사이트 바 (중간 회색)
`[F 로고]  Inbox │ My Tasks │ Media │ Projects▾ │ All Pages▾ │ People │ Apps▾ │ Resource Planning`
… 우측: `🔍 Search Site…  ＋  🔔  ⚙  [아바타]`
- 현재 브랜딩 로고는 **초록색 "F"** (Flow Production Tracking).

### 2단: 프로젝트 엔티티 탭 바 (거의 검정)
`CTR │ Overview │ Project Details │ Assets │ Shots │ Tasks │ Notes │ Versions │ Playlists │ Other▾ │ Project Pages▾`
… 우측: `Project Actions▾`
- **이게 내 클론의 좌측 사이드바를 대체해야 한다.** 프로젝트의 엔티티 페이지가 가로 탭.
- 활성 탭은 하단 **teal 밑줄**.

### (옵션) 파란 어드민 공지 배너
상단에 dismiss(X) 가능한 정보 배너.

## 2. 엔티티 그리드 (Shots/Assets/Versions 공통)

### 2.1 페이지 타이틀 줄
`Shots ☆` (즐겨찾기 별). 우측에 페이지 액션 아이콘들.

### 2.2 그리드 툴바 (좌→우)
`[▤ 리스트][▦ 썸네일]  [＋Add Shot ▾(파란 버튼)]  ↕Sort▾  ⊞Group▾  Fields▾  More▾  Pipeline▾`
… 우측: `🔍 Search Shots…   ▢레이아웃   ▼Filter   ⟳새로고침`
- **Add <Entity>** 는 채도 높은 **파란 버튼**(+드롭다운 캐럿).
- Sort/Group/Fields/More/Pipeline = 드롭다운 텍스트 버튼.

### 2.3 컬럼 (스프레드시트, 매우 많음)
Shots 예시 헤더:
`☐ │ Status │ Cut O… │ Reel │ Sequence │ Thumbnail │ Shot Code │ Description │ Work Guide │ Assets │ Priority │ Camera │ Lens │ Cut In │ Cut Out │ Cut Dura… │ Type │ Tags │ Note`
- **맨 앞 체크박스 열** + **Status가 첫 데이터 열(좁은 색상 셀)**.
- 활성 정렬 컬럼 헤더는 teal 하이라이트.
- **대부분 컬럼이 비어 있음** — 빈 칸이 많은 게 원본의 실제 모습(밀도 높은 표).

### 2.4 셀 디테일
- **Status**: 텍스트 pill이 아니라 **셀 전체가 상태색**으로 칠해지고 작은 아이콘(○).
- **Thumbnail**: 이미지가 있으면 작은 썸네일(+▶ 재생 오버레이), 없으면 "Upload Thumbnail" 링크.
- **Shot Code / Reel / Sequence / Link**: teal **하이퍼링크** 텍스트(+Sequence엔 상태 점).
- 행 높이: 텍스트行은 조밀, 썸네일行은 약간 큼.

### 2.5 하단 바
`1 - 11 of 11 Shots` (가운데) … 우측 `100 per page`.

## 3. 디테일 페이지 (`/detail/Shot/<id>`)

- **브레드크럼**: `TST > TST_0020 ▾`  +  `← →` 이전/다음.  우측 `Follow │ 📄 │ ⟳`.
- 좌측 **큰 썸네일**(약 250px 정사각, 없으면 필름 아이콘 placeholder).
- 우측 **필드 표**(라벨–값): Shot Code / Sequence(상태점) / **Status(넓은 색상 셀)** / Description / Type …
- 큰 **엔티티 타이틀**(TST_0020).
- 하단 **연관 탭**: `Activity │ Shot Info │ Tasks │ Notes │ Versions │ Publishes │ Assets │ History`.
  각 탭은 **자체 서브 그리드 + 툴바**(Add Version 파란버튼/Sort/Group/Fields/More, Latest 체크, Search, Filter).
- 빈 상태 메시지: "Once you create some Versions, this is where you can see them." + 파란 CTA.

> 내 클론은 디테일을 **사이드 패널**로 처리 → 원본은 **전체 페이지**. 이 구조 차이가 큼.

## 5. 내 클론이 바꿔야 할 것 (우선순위)

1. **좌측 사이드바 → 상단 2단 네비**(글로벌 바 + 프로젝트 엔티티 가로 탭). 🔴 구조
2. **그리드 툴바를 원본대로**: 뷰토글 │ 파란 Add 버튼 │ Sort/Group/Fields/More/Pipeline │ Search │ Filter. 🔴
3. **Status를 첫 컬럼의 색상 셀**로(+체크박스 열). 🔴
4. **컬럼 수 확대 + 빈칸 허용**(스프레드시트 밀도). 🟡
5. **디테일을 전체 페이지로**(브레드크럼+큰 썸네일+필드표+연관 탭 서브그리드). 🔴
6. **하단 페이지네이션 바**(`N of M`, `100 per page`). 🟡
7. **링크 텍스트 teal**, 테마 약간 밝게. 🟡
8. 썸네일 셀에 "Upload Thumbnail" placeholder + ▶ 오버레이. 🟢

## 5-b. 코드(CSS) 직접 추출 — 정확한 값

스크린샷 눈대중 대신 **실제 `base.css`를 내려받아 정확한 값을 추출**했다.
(추출본: [`이미지/원본참고/base_css_추출.txt`](이미지/원본참고/base_css_추출.txt))

| 항목 | 실측값 |
|------|--------|
| 폰트 | `ArtifaktElement` / `proxima-nova`, helvetica, arial (라이선스 폰트→fallback) |
| 주 색상 | `--sg_PRIMARY_COLOR: #3c3c3c` |
| 액센트(활성/선택) | **#0696d7** (Autodesk 블루) |
| 그리드 셀 배경(다크) | `.sg_cell` → **#222 / #3c3c3c** |
| 그리드 셀 배경(라이트) | #f5f5f5 / #fff |
| 보더 | #4d4d4d | 텍스트 #e2e2e2, 흐림 #999 |
| 글로벌 네비 테마 | theme1 darkBlue `#0c2c54`, theme2 green … (선택형) |
| 디자인 시스템 | `--sgds_basics_colors_*` (ShotGrid Design System 토큰) |

→ 내 클론의 토큰을 **추측값에서 위 실측값으로 교체**했다(`styles.css :root`).
프레임워크는 **ExtJS**(`ext_all.css`) 기반임도 확인(직접 의존하진 않음 — 동일 룩을 자체 구현).

> **주의(IP)**: 원본의 독점 CSS/JS 번들을 통째로 복사하지 않는다. **정확한 디자인 값
> (색·크기·구조)만 추출해 자체 코드로 재구현**한다.

## 6. 비고 — 파이프라인 스텝 컬럼
이 프로젝트의 기본 Shots 페이지에는 파이프라인 스텝 요약 컬럼이 **표시되어 있지 않았다**
(Fields로 추가하는 선택 컬럼). 즉 내 1차의 "파이프라인 스트립"은 유효한 원본 기능이나
**기본 화면은 아니다**. 2차에서는 선택적으로 켜는 컬럼으로 두는 게 더 정확하다.
